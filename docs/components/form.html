Select Accessibility

  • Always pair with a <label> using matching for and id attributes
  • Use meaningful option text, not just codes
  • Provide a default "Choose..." option when selection is required
  • Use .form-help for additional context
  • Use .form-error with .error class to indicate validation errors

When to Use Select

  • 5-15 options that users need to choose from
  • Options are mutually exclusive (only one can be selected)
  • Space is limited and you need a compact control

When Not to Use Select

  • 2-4 options: Use radio buttons instead for better visibility
  • More than 15 options: Consider autocomplete or search
  • Multiple selections needed: Use checkboxes or a multi-select pattern

Checkbox Accessibility

  • Use <fieldset> and <legend> to group related checkboxes
  • Each checkbox must have a unique id and matching for attribute on the label
  • Use clear, descriptive labels that explain what checking the box means
  • Place the label after the checkbox for better clickability
  • Use .form-help for additional context
  • Use .form-error with .error class to indicate validation errors

When to Use Checkbox

  • Users can select zero, one, or multiple options from a list
  • Each option is independent (selecting one doesn't affect others)
  • Binary choices (on/off, yes/no, agree/disagree)

When Not to Use Checkbox

  • Mutually exclusive options: Use radio buttons instead
  • Immediate action required: Use a toggle switch
  • More than 10 options: Consider a multi-select dropdown

Radio Button Accessibility

  • Always use <fieldset> and <legend> to group radio buttons
  • All radio buttons in a group must share the same name attribute
  • Each radio button must have a unique id and matching for attribute on the label
  • Use clear, descriptive labels that make options easy to distinguish
  • Place the label after the radio button for better clickability
  • Pre-select a sensible default option when appropriate
  • Use .form-help for additional context
  • Use .form-error with .error class to indicate validation errors

When to Use Radio Button

  • 2-5 mutually exclusive options where users must choose exactly one
  • All options should be visible without scrolling
  • Choices are equally important and need equal visual weight

When Not to Use Radio Button

  • Only 2 options: Consider a checkbox or toggle instead
  • More than 5 options: Use a select dropdown to save space
  • Multiple selections allowed: Use checkboxes instead
