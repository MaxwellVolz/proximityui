Accessibility

  • Use semantic HTML elements inside cards (<article>, <section>, etc.)
  • Ensure headings follow a logical hierarchy (don't skip levels)
  • If the card is clickable, wrap it in a proper link or button element
  • Ensure sufficient color contrast for all text content
  • Make interactive elements keyboard accessible with proper focus states
  • Consider using aria-labelledby to associate card titles with their content

When to Use

  • Group related information and actions together
  • Create visual separation between different content sections
  • Display summaries or previews that link to detailed views
  • Build dashboards with modular, scannable content blocks
  • Present options or choices to users (e.g., pricing plans, product features)

When Not to Use

  • For simple text blocks: Use regular HTML elements instead
  • When content doesn't need visual separation: Avoid unnecessary boxing
  • For primary page navigation: Use navigation menus instead
  • In dense data tables: Cards add too much visual weight
  • For inline content within paragraphs: Use appropriate inline elements
