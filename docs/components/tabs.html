Note: Tabs work without JavaScript using anchor links and the :target pseudo-class. proximity.js progressively enhances with keyboard navigation and ARIA support.

HTML Structure

The basic HTML structure for tabs:

<div class="tabs">
  <div class="tab-list" role="tablist">
    <a href="#panel-1" class="tab">Tab 1</a>
    <a href="#panel-2" class="tab">Tab 2</a>
    <a href="#panel-3" class="tab">Tab 3</a>
  </div>
  <div id="panel-1" class="tab-panel">
    <!-- Panel 1 content (visible by default) -->
  </div>
  <div id="panel-2" class="tab-panel" hidden>
    <!-- Panel 2 content -->
  </div>
  <div id="panel-3" class="tab-panel" hidden>
    <!-- Panel 3 content -->
  </div>
</div>

Keyboard Navigation

When JavaScript is enabled, tabs support full keyboard navigation:

Key Action
Tab Moves focus into or out of the tab list
→ Arrow Right Moves focus to next tab, wraps to first
← Arrow Left Moves focus to previous tab, wraps to last
Home Moves focus to first tab
End Moves focus to last tab

Note: Arrow keys automatically activate the focused tab (automatic activation pattern).

Accessibility

  • ARIA Roles: The component uses role="tablist", role="tab", and role="tabpanel" for proper semantics.
  • ARIA Attributes: JavaScript manages aria-selected, aria-controls, and aria-labelledby attributes.
  • Focus Management: Only the active tab has tabindex="0", inactive tabs have tabindex="-1".
  • Keyboard Support: Full keyboard navigation with arrow keys, Home, and End keys.
  • Progressive Enhancement: Works without JavaScript using anchor links and the :target pseudo-class.
  • Focus Indicators: Visible focus rings appear when navigating with keyboard (:focus-visible).

When to Use

  • Organizing related content that doesn't need to be visible simultaneously
  • Settings or configuration interfaces with multiple categories
  • Product details with different aspects (overview, specs, reviews)
  • Single-page applications with distinct views
  • Dashboard interfaces with different data visualizations
  • When screen space is limited and you need to reduce scrolling

When Not to Use

  • Sequential processes: Use a stepper or wizard component instead
  • Comparison tasks: When users need to compare content side-by-side
  • Navigation between pages: Use a navigation menu instead
  • Single piece of content: Don't use tabs for just one or two items
  • Collapsible content: Use accordion or <details>/<summary> if all content should be accessible at once
  • Critical content: Important information should be immediately visible, not hidden in tabs
