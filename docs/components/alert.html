Accessibility

  • Use appropriate semantic HTML inside alerts (headings, paragraphs, lists)
  • Consider using ARIA role="alert" for time-sensitive messages that should interrupt the user
  • Use role="status" for less urgent informational messages
  • Ensure sufficient color contrast between text and background
  • Don't rely solely on color to convey meaning - use text labels like "Success:", "Warning:", etc.
  • If dismissible, ensure the close button is keyboard accessible

When to Use

  • Display system messages or feedback after user actions
  • Show important warnings that require user attention
  • Communicate errors that occurred during form submission or operations
  • Provide context or additional information about page state

When Not to Use

  • Time-sensitive notifications: Use toast/snackbar instead
  • Inline validation errors: Use form error messages instead
  • Permanent page content: Use regular content blocks
  • User-dismissible temporary messages: Consider toast patterns
