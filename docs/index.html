Proximity UI

A small, CSS-first component system for server-rendered applications.
Behavior lives close to markup. Semantics come first. JavaScript is optional.

View on GitHub
No build step required. Just drop in proximity.css and start building. Works great with HTMX, but doesn't depend on it.

Design Philosophy

Readable HTML

Semantic markup that's easy to understand and maintain

No Build Step

Direct CSS/JS files you can use immediately

Copy-Paste

Components you can grab and customize

Progressive

Works without JavaScript, better with it

Goals

  • Readable HTML
  • No build step
  • Copy-paste components
  • Progressive enhancement
  • Long-term maintainability

Anti-Goals

  • JavaScript frameworks
  • Utility-first CSS
  • Component compilers
  • Client-side state management

Why Proximity UI?

Server-First

Built for server-rendered applications. Pairs naturally with HTMX for dynamic interactions without complex JavaScript.

CSS Variables

Design tokens as CSS custom properties. Customize colors, spacing, and typography without touching the source.

Accessible

ARIA roles, keyboard navigation, and focus management built-in. Screen reader friendly from the start.

Dark Mode

Automatic dark mode via prefers-color-scheme. No toggle needed, respects system preferences.

Lightweight

Small footprint with proximity.js under 5kb gzipped. Only loads what you need, when you need it.

Future-Proof

No framework lock-in, no CLI tools, no breaking changes. Built on web standards that will work for years.

Get Started

Quick Start

Add Proximity UI to your project in seconds. No npm, no build tools, just HTML and CSS. 

<!-- Add to your <head> -->
<link rel="stylesheet" href="proximity.css">
<script src="proximity.js"></script>

<!-- Optional: Add HTMX for dynamic interactions -->
<script src="https://unpkg.com/htmx.org@2.0.4"></script>

<!-- Start using components -->
<button class="btn primary">Click me</button>

Local Development

View the documentation locally with no setup required: 

python3 -m http.server
# Open http://localhost:8000/docs
